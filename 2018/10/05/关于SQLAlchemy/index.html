关于SQLAlchemy

发表于 2018-10-05 | 分类于 技术篇 | 阅读次数:
1.SQLAlchemy简介
SQLAlchemy是一个与数据库进行交互的的库，它可以让你用python的类和语句创建数据模型来查询数据。它可以连接许多数据库，包括Postgres、MySQL、SQLite和Oracle等等。

使用SQLAlchemy时，我们可以不必关注不同数据库的语法，SQLAlchemy相当于提供了一个统一的接口，可以让你从各种数据库语言中解放出来。

SQLAlchemy通过两个主要的模型来实现灵活的操作：SQLAlchemy Core和SQLAlchemy ORM。

SQLAlchemy Core是以模式为中心，和普通SQL一样有表，键和索引等。SQLAlchemy Core最擅长的是数据仓库、报表分析、以及其他使用数据查询和其他操作可以牢牢掌握的地方。它拥有强大的数据库连接池和数据结果集的优化，非常适合处理大量数据，甚至多数据库也适用。

但是，如果你更侧重领域驱动设计，那么SQLAlchemy ORM就可以将原数据和业务对象的底层的模式和结构大部分细节都封装起来。这样封装让数据库连接更简单，更像Python代码。

2.SQLAlchemy连接数据库
连接数据库需要SQLAlchemy引擎，SQLAlchemy引擎为数据库执行SQL语句创建了一个常用的接口。引擎通过封装一个数据库连接池和方言来实现不同数据库类型统一的接口，这样做使得python代码不需要关心不同数据库DBAPI之间的差异。SQLAlchemy提供了一个带连接字符串和一些参数的函数来创建引擎。

字符串格式如下：
<数据库类型>+<数据库驱动>://<用户名>:<密码>@<主机名或IP>:<端口号>/数据库名称

例如：
mysql+mysqlconnector://root:1234567@localhost:3306/MyDatabase

然后我们通过创建引擎实例来连接一个已知数据库：

create_engine函数创建了一个引擎实例，但是它并没有真正连接，直到一个动作要求引擎执行时才会执行，比如查询或者新建数据。

create_engine函数的可选参数有：

echo：表示引擎行为的日志是否显示，默认是False。
ecoding：编码，默认是utf-8。
isolation_level：指定隔离级别，包括READ_COMMIT，READ_UNCOMITTED，REPEATABLE READ，SERIALIZABLE，AUTOCOMMIT。
pool_recyle：指数据库连接多少秒循环检查一次，默认值为-1，表示没时间限制，一直连接。但MySQL默认超过8小时空闲会断开连接，为了绕开这个问题，可以设置为pool_recyle=3600，表示每一小时自动连接一次。

引擎建立后，通过connect()函数就可以连接数据库。

3.1 SQLAlchemy的数据类型

SQLAlchemy有四种数据类型，这里主要掌握通用类型，四种数据类型如下：
1.通用型（Generic）
2.SQL标准类型（SQL standard）
3.数据库特有类型（Vendor Specific）
4.用户自定义类型（User Defined）

SQLAlchemy为不同的数据库定义了一些通用的数据类型。这些类型都在sqlalchemy.types模块中，为了引用方便也支持放在sqlalchemy里面。

3.2 Metadata

Metadata表示元数据，我的理解是把它当成一个容器，用来盛放数据的信息，这些信息包括表的结构，如表名、字段等，还包括引擎和连接信息。在定义表时，将表结构放入metadata，最后通过连接引擎在数据库里创建定义好的表。

3.3 Table和Column

使用Table定义表结构，包括表名、存放的Metadata对象和各个字段信息。Column用于定义表中的各个字段，包括字段名、数据类型。还可以通过一些参数进行其他设置，比如使用primary_key把某一字段设置为主键，nullable表示是否为空，表示值是否唯一，default可定义初始值。

定义完这些之后，用metadata的create_all方法就可以把表持久化到数据库中。默认情况下，create_all方法不会重新建立已经存在的表，即表名相同的表，所以运行多次也很安全。

4. SQLAlchemy ORM

SQLAlchemy作为一种对象关系映射模型，那么它到底如何映射的？在SQLAlchemy中我们只需要写python语言，通过这个ORM，把python语言转化成数据库语言，从而实现对数据库的操作，它们的对应关系如下。
python的类 –> 数据库的表
python类的属性 –> 表中的字段
python类的实例 –> 表中每一行的内容
