Python中的命名规则

发表于 2018-11-16 | 分类于 技术篇
命名约定

1.用单下划线( _ )开头的表示模块变量或函数是被保护的（即使用 import * from 时不会包含）。

2.用双下划线( __ )开头的实例变量或方法表示类内私有，从外部无法获取。

3.对类名使用大写字母开头的单词（如 CapWorld ），模块名应该用小写加下划线的方式（如 lower_with_under.py）。普通变量用小写字母加下划线方式，全局变量使用大写字母加下划线（如S OME_VARIABLE）。

应该避免

1.除了计数器和迭代器外，避免单字符名称。

2.避免以双下划线开头以双下划线结尾的变量，这类变量为 python 专有变量，如 name, doc 。

3.因为 python 是动态类型语言，变量名不应带有类型信息，类似 student_dict, name_list 等不是好的命名方式。
